#### 89. Які типи асинхронних операцій здатен виконувати Node.js?

Node.js здатен виконувати декілька типів асинхронних операцій, серед яких:

1. **Асинхронні введення/виведення (I/O) операції**:
   - Node.js спеціалізується на асинхронних I/O операціях, таких як зчитування та запис файлів, робота з базами даних та мережевими з'єднаннями. Асинхронні I/O операції дозволяють обробляти інші запити, не чекаючи завершення попередньої операції.

2. **Таймери та планування виконання**:
   - `setTimeout()` та `setInterval()` є прикладами асинхронних операцій, де певні коди виконуються через встановлений проміжок часу або в певних інтервалах.

3. **Запити до зовнішніх API**:
   - Використання HTTP або інші запити для зв'язку з зовнішніми API є поширеними асинхронними операціями в Node.js.

4. **Операції з "Worker Threads"**:
   - З версії 10.5.0 Node.js підтримує "Worker Threads" для асинхронного виконання задач, які вимагають багато обчислень, в окремих потоках виконання.

5. **Події системи**:
   - Node.js постійно прослуховує події системи через внутрішню систему обробки подій (Event Loop), що дозволяє організовувати асинхронні виклики на основі різноманітних подій.

Усі ці типи операцій допомагають Node.js ефективно обробляти асинхронні запити, що особливо корисно для серверних додатків, які мають працювати зі значною кількістю одночасних з'єднань.
