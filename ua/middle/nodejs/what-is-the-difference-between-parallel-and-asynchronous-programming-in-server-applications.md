#### 88. У чому полягає різниця паралельного та асинхронного програмування на прикладі серверних застосунків?

І в паралельному, і в асинхронному програмуванні ми прагнемо поліпшити продуктивність та швидкодію додатків, але підходи до цього різні.

### Асинхронне програмування

Асинхронне програмування дозволяє виконувати різні операції незалежно, не блокуючи основний потік виконання програми. Наприклад, у Node.js використовується модель подій та неблокуючий I/O, що дозволяє обробляти запити, не чекаючи завершення кожного з них. Це досягається через використання зворотних викликів (callbacks), промісів (promises) або `async/await`. 

**Приклад:**
```javascript
const fs = require('fs');

fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

console.log('Цей код виконується до того, як файл прочитаний');
```
У цьому прикладі файл прочитується асинхронно, тому повідомлення "Цей код виконується до того, як файл прочитаний" виведеться до завершення операції читання файлу.

### Паралельне програмування

Паралельне програмування передбачає виконання декількох операцій одночасно за допомогою мультизадачності (наприклад, через різні потоки або процеси). У випадку Node.js це менш поширено, оскільки Node.js використовує однонитну модель. Однак через кластеризацію або за допомогою використання робітників (worker threads) можна досягти паралельного виконання.

**Приклад:**
```javascript
const { Worker } = require('worker_threads');

const worker = new Worker(`console.log('Цей код виконується у іншому потоці');`, { eval: true });

worker.on('exit', () => console.log('Робітник завершив свою роботу'));

console.log('Цей код виконується у головному потоці');
```
У цьому прикладі одночасно виконуються два блоки коду в різних потоках.

### Висновки

- **Асинхронність** підходить для великої кількості незалежних I/O операцій. Чудово підходить для серверів, що обробляють багато запитів одночасно, завдяки неблокуючій обробці.
- **Паралельність** забезпечує одночасність виконання чисельних завдань і корисна при важких обчисленнях, які можуть бути виконані незалежно один від одного.

Обидва підходи можуть взаємодіяти одне з одним, і часто застосовується комбінація цих методів для максимальної продуктивності та ефективності.
